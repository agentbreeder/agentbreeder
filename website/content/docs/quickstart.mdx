# Quickstart

Deploy your first agent in under 5 minutes.

---

## Prerequisites

| Requirement | Min version | Why |
|-------------|-------------|-----|
| **Python** | 3.11+ | CLI, API server, Python SDK |
| **Docker Desktop** | 24+ | Local deploys and container builds — must be **open and running** |
| **Node.js + npx** | 18+ | MCP servers (run via `npx`) |

<Callout type="info" title="Just starting out?">
  Python 3.11+ and Docker Desktop are the only hard requirements for this quickstart. Node.js is only needed when you wire in MCP servers.
</Callout>

Check yours:

```bash
python3 --version   # need 3.11+
docker info         # must succeed — if it errors, open Docker Desktop first
```

<Callout type="warn" title="Docker not running?">
  Open the **Docker Desktop** app before running any `docker` commands. On Linux: `sudo systemctl start docker`. Verify with `docker info` — a successful response means Docker is ready.
</Callout>

---

## 1. Install

Pick the method that matches your environment:

| Method | Command |
|--------|---------|
| **pip** (Python 3) | `pip3 install agentbreeder` |
| **Homebrew** (macOS) | `brew tap agentbreeder/agentbreeder && brew install agentbreeder` |
| **Docker** | `docker run rajits/agentbreeder-cli --help` |
| **npm** | `npm install @agentbreeder/sdk` |

```bash
pip3 install agentbreeder
agentbreeder --help
```

> **`agentbreeder: command not found` after install?** pip's script directory may not be in your PATH. Run:
> ```bash
> python3 -c "import sysconfig; print(sysconfig.get_path('scripts'))"
> ```
> Add that path to your `PATH` (e.g. in `~/.zshrc`), then open a new terminal. See [full troubleshooting →](how-to#agentbreeder-command-not-found)

> On macOS, **Homebrew is the easiest install** — it handles PATH automatically.

All install options → [How-To: Install →](how-to#install-agentbreeder)

---

## 2. Create Your First Agent

```bash
agentbreeder init
```

The wizard asks 5 questions:

| # | Question | Example |
|---|----------|---------|
| 1 | Framework | LangGraph |
| 2 | Cloud target | Local |
| 3 | Agent name | `support-agent` |
| 4 | Team | `engineering` |
| 5 | Owner email | `alice@company.com` |

Generated project:

```
support-agent/
├── agent.yaml          # The only file AgentBreeder needs
├── agent.py            # Working agent code for your framework
├── requirements.txt
├── .env.example
└── README.md
```

---

## 3. Validate

```bash
cd support-agent
agentbreeder validate agent.yaml
```

Runs three checks: YAML syntax → JSON Schema → semantic validation (framework supported, team exists).

---

## 4. Deploy Locally

```bash
agentbreeder deploy agent.yaml --target local
```

The 8-step atomic pipeline:

```
✅  YAML parsed & validated
✅  RBAC check passed
✅  Dependencies resolved
✅  Container built (langgraph runtime)
✅  Deployed to Docker Compose
✅  Health check passed
✅  Registered in org registry
✅  Endpoint returned
```

If any step fails, the entire deploy rolls back.

---

## 5. Verify

```bash
agentbreeder status           # see your agent's endpoint and health
agentbreeder logs support-agent --follow   # tail live logs
agentbreeder chat support-agent            # interactive chat
```

That's it — your agent is live.
